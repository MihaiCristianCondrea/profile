Prepare for Chrome on Android going edge-to-edge

Chrome 135 introduces dynamic edge-to-edge browsing on small-screen Android devices. The viewport can now extend underneath Android's gesture navigation bar, while Chrome adds a transient "chin" that slides away as users scroll. This page summarizes what the change means for this website and links to the latest guidance so every surface stays polished when Chrome starts drawing behind the system bars.

Site readiness checklist

The site already ships viewport-fit=cover so content can expand into the full viewport. To align with the new Chrome defaults we have tightened up our safe-area handling and recommend the following checks whenever you add new components:

  • Size any bottom-anchored UI (drawers, banners, FAB rails) using the calc(env(safe-area-inset-bottom, 0px) - var(--safe-area-max-inset-bottom)) pattern. This allows Chrome's performance fast-path to kick in when the chin moves and keeps animations stutter-free.
  • Allow content backgrounds to bleed into the navigation area by padding elements with var(--safe-area-max-inset-bottom). Avoid recomputing layout by not applying the dynamic env(safe-area-inset-bottom) directly as padding.
  • When embedding media or dialogs that sit flush with the edges, remember to apply the existing CSS custom properties (--safe-area-inset-*) so nothing is obscured as Chrome animates its bars.

Test edge-to-edge before it launches

You can validate layouts early by enabling the relevant feature flags in Chrome Canary or Dev on Android. Toggle the following entries under chrome://flags, then restart Chrome twice to ensure they stick:

  • EdgeToEdgeBottomChin ("Enabled Debug" helps visualize the chin)
  • DrawCutOutEdgeToEdge
  • BottomBrowserControlsRefactor → Enabled Dispatch yOffset
  • DynamicSafeAreaInsets and DynamicSafeAreaInsetsOnScroll
  • EdgeToEdgeWebOptIn, DrawKeyNativeEdgeToEdge
  • EdgeToEdgeSafeAreaConstraint → Enabled Scrollable Variation
  • Leave DrawNativeEdgeToEdge and EdgeToEdgeEverywhere disabled

With these flags active, scroll through every page, especially those that pin controls to the bottom edge, and confirm that content remains tappable as the chin hides and reveals itself.
